## 1. 부모(마스터) 모드 — 동작

- **진입**: 제목 "별 사냥 ☄️ 스도쿠"를 ~0.8초 **길게 누르기**(터치/마우스 모두). 짧은 탭은 무동작, 길게 눌러도 컨텍스트 메뉴 안 뜨게 처리. 화면에 노출 버튼 없음(아이 눈에 안 띄게).
- **3가지 진입 상태**:
  - (a) 비번 미설정 → **"부모 비밀번호 만들기"**(4자 이상 + 두 칸 일치 검증)
  - (b) 설정됨 & OFF → **"부모 비밀번호 입력"**(맞으면 ON) + **"비밀번호 초기화"**(아래)
  - (c) ON → **메뉴**: 여기까지 열어주기 / 부모 모드 끄기 / 비밀번호 변경 / 닫기
- **여기까지 열어주기**: 부모 모드에서 원하는 단계·난이도를 먼저 고른 뒤 메뉴에서 누르면 `save.maxLevel = 현재 ui.level`(상향만) 영구 저장. 이후 부모 모드를 꺼도 아이가 그 단계까지 풀 수 있고, frontier에서 50문제(`LEVEL_TARGET`) 풀면 정상적으로 다음이 열림.
- **부모 모드 끄기**: `masterOn=false`. 현재 선택 레벨이 잠긴 레벨이면 `maxLevel`로 되돌려 새 문제 생성.
- **비밀번호 변경**(메뉴): 이미 인증됐으므로 새 비번 2번 입력으로 변경.
- **비밀번호 초기화**(입력 화면): 한 번 더 눌러 확인 → `masterPw`만 삭제(**점수·도장·단계는 유지**) → 새로 만들기. 비번을 잊어도 앱 데이터 삭제 없이 복구 가능.
- **기록 미반영**: 부모 모드 중 `onSolve` 진입부에서 분기 → 별가루 효과만, 점수/별/도장/연속/레벨업·`persist()` 모두 건너뜀.

## 2. 코드 변경 (`index.html`)

- `let masterOn=false`(세션 한정, 저장 안 함), 비번 헬퍼 `encPw`(btoa 가벼운 난독화)·`hasMasterPw`·`checkPw`.
- `save` 기본값에 `masterPw:""` 추가 — `Object.assign({기본값}, 저장값)` 덕에 **기존 저장본과 하위호환**(기록 보존).
- `renderSelectors`의 잠금 계산에 `masterOn||…` 반영. `chooseStage`는 부모 모드면 easy부터 시작.
- 사이드바에 "🔓 부모 모드 (끄기)" 배지(`renderStrip`), 클릭 시 메뉴.
- 새 섹션 **7.5) 부모 모드**: `openMaster`/`mMenu`/`mInput`/`mCreate`/`mChange`/`savePw`/`submitMaster`/`resetPw`/`openUpTo`/`toggleMasterOff`/`bindLongPress`.
- CSS: `#master` 모달(`#win` 패턴 복제), input 스타일(앱 최초 input), 에러 텍스트, `.master-badge`.

## 3. 배포 / 캐시 (`sw.js`)

- 캐시 버전 `byeolsudoku-v1` → **`v3`**(로컬 테스트 중 v2 거쳐 v3). 변경 배포 반영을 위해 필수.
- **로컬 테스트 메모**: 서비스워커가 cache-first라, 코드 수정 후 새 버전이 안 보이면 `Ctrl+Shift+R`(강력 새로고침) 또는 DevTools → Application → Service Workers → "Bypass for network". 갤럭시탭은 버전 올려 배포 후 **앱 닫았다 (보통 두 번) 켜면** 자동 갱신.

## 4. 데이터 보존 원칙 (향후 모든 수정에 적용)

- 진행기록(도장·점수·단계)은 코드/GitHub가 아닌 **태블릿 localStorage**(`byeolsudoku_save_v2`)에 저장 → 재배포해도 유지.
- 반드시: 저장 키 불변 + 데이터는 **필드 추가만**(삭제·리네임 금지). 구조 대변경 시 마이그레이션 코드 동봉.
- 데이터가 지워지는 경우: PWA 삭제·재설치, "데이터/저장공간 삭제"(≠ 캐시 삭제).
