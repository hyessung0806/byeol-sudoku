## 오늘 한 일 요약 (2건)
1. **가로 레이아웃 사이드바 위치 정리** — 왼쪽 위에 붙어 보드와 어긋나던 사이드바를 보드와 세로 중앙으로 맞추고, 둘이 너무 벌어지지 않게 묶음.
2. **세로모드 허용** — 설치된 PWA가 가로로만 고정되던 것을 풀어 세로/가로 둘 다 동작하게 함.

> **최종 결론 먼저:** 갤럭시탭 설치본에서 ① 가로일 때 사이드바가 보드와 눈높이가 맞고 가깝게 붙어 보이며, ② 세로로 들어도 위→아래 1열로 스택되어 정상 작동한다. 변경은 정적 3파일에만 국한(`index.html` 가로 미디어쿼리, `manifest.json` orientation, `sw.js` 캐시버전 v3→v4). 세로용 CSS는 새로 만들지 않았다 — 기존 기본(모바일) 1열 레이아웃이 그대로 세로를 처리하므로 manifest의 방향 고정만 풀면 됐다.

## 1. 가로 사이드바 위치 (index.html)
`@media (min-width:820px) and (orientation:landscape)` 블록만 수정:
- `.sidebar`: `justify-content:flex-start` → **`center`** (사이드바 내용을 세로 중앙으로 내려 보드와 눈높이 맞춤) — **이게 핵심 수정.**
- `.main`: `flex:1 1 auto` → **`flex:0 1 auto`** (본문이 남은 폭을 다 먹지 않게 해, `.layout`의 `justify-content:center`가 [사이드바+보드] 묶음을 중앙에 모음 → 둘 사이 간격 축소)
- `.layout` `gap:22px` → **`36px`** (붙인 뒤 답답하지 않게 간격 확보)

세로/모바일 기본 레이아웃은 손대지 않음 → 영향 없음.

## 2. 세로모드 허용 (manifest.json)
- `"orientation": "landscape"` → **`"any"`**. 설치 PWA가 가로/세로 모두 허용.
- 세로일 땐 기본 1열 스택(`--bw / --panel: min(92vw,460px)`)이 그대로 적용됨.

## 3. 캐시 버전 (sw.js) — 규칙상 필수
- `byeolsudoku-v3` → **`v4`**. 안 올리면 태블릿이 옛 화면을 캐시.

- 배포 후 갤럭시탭 확인 필요: 앱 완전 종료 후 재실행(또는 캐시 삭제)으로 새 버전 로드.

## 2차 작업 — 사이드바 상단정렬·간격·글씨 (같은 날 추가)
1차 배포 후 노트북(가로)에서 확인하니 사이드바가 세로 중앙이라 보드와 눈높이가 안 맞고, 간격이 과하게 넓으며, 글씨가 작다는 피드백. 가로 미디어쿼리만 재작성:
- **상단 정렬**: `.layout align-items: stretch → flex-start` (사이드바 제목 top과 보드 top을 같은 라인에). `.sidebar`의 `justify-content:center` 제거, `.main`의 `justify-content` 제거.
- **간격 과대 원인 해결**: `.main`이 base의 `width:100%`를 그대로 들고 있어 보드가 넓은 영역 안에서 가운데로 밀려 큰 여백이 생겼던 것. 미디어쿼리에서 **`.main { flex:0 0 auto; width:auto }`** 로 보드 크기에 맞춰 줄임. `gap:36px → 24px`.
- **사이드바 글씨 확대**(가로 전용, 세로/모바일 영향 없음): `.sidebar h1` 24→30px, `.tab` 14→16px, `.strip` 14→16.5px, `.mission` 13→15px, `.label`/`.pill`/`.ptext` 12~13→15px. 사이드바 폭 `340 → 380px`(글씨 키워도 단계/난이도 pill이 안 줄바꿈되게).
- 캐시 `v4 → v5`.

> 교훈: base `.main { width:100% }`가 미디어쿼리의 `flex:0 1 auto`를 무력화(flex-basis:auto가 width:100%로 해석)해 보드가 가운데로 떠 있었음. 가로 레이아웃에서 본문 폭을 줄이려면 `width:auto`를 반드시 같이 줘야 함.
